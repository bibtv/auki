## Integration Points Summary

| ROS Component | Connection to Bridge | Data Exchanged |
|--------------|---------------------|----------------|
| **TF** | `tf_buffer_.lookupTransform()` | Robot pose (world → base_link) |
| **Odometry** | `/odom` subscriber | Backup pose source |
| **Navigation** | `cmd_vel` monitored | Awareness of other robots prevents conflicts |
| **Sensors** | Optional: share lidar/camera via domain | Enhanced perception |
| **State Machine** | Read-only awareness | Know where other robots are |

## What Posemesh Enables for Delivery Robots

| Capability | Without Posemesh | With Posemesh |
|------------|------------------|---------------|
| **Multi-robot coordination** | Ad-hoc communication | Shared spatial awareness |
| **Collision avoidance** | Local sensors only | Know robot positions globally |
| **Fleet management** | Central server required | Decentralized discovery |
| **AR human guidance** | Not possible | Show robot to user via phone |
| **域定位** | SLAM required | Pre-mapped domain coordinates |
